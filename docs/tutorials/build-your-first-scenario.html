1 · The big picture

This tool is a dynamic partial-equilibrium model of the emission-allowance market. Each year it finds the single carbon price at which the market clears:

net allowance demand (across all firms)  =  allowances supplied (auction + carry)

Every firm reacts to that price by abating up to the point where its marginal abatement cost equals the price, then buying or selling the rest. Everything else — output, energy prices, the macro-economy — is an input. As you go down this tutorial you'll progressively let the market talk back to those inputs:

Base
Fixed cap, exogenous baseline emissions. The classic clearing model. (Step 1)
Mechanisms
Supply-side policy on top of the cap: MSR reserve, or the Taylor-rule Carbon Cap Rule. (Steps 2–3)
Feedback A
Carbon-intensive activity contracts as the price rises, solved inside clearing. Still partial equilibrium. (Step 4)
Feedback B
Iterate the market to a joint equilibrium with an external energy / CGE / DSGE model. (Step 5)

2 · Setup & how to run

From the repository root, install the dependencies once:

pip install -r requirements.txt      # numpy, pandas, scipy, matplotlib

A scenario is a single .json file. Run one and print the results table with:

python ets_framework.py --config path/to/your_scenario.json

Or open the browser-based editor to build and run scenarios visually:

python ets_framework.py --gui          # opens the scenario editor
TipEvery bundled example lives in examples/. List the built-in sample modes with python ets_framework.py --list-modes, or just point --config at any example file.

1Your first scenario

Save this as my_first_scenario.json. It's one year, a fixed cap of 300 Mt, and two firms — a cheap-to-abate Power sector and a costlier Steel sector. Each has a linear marginal abatement cost (MAC) curve.

{
  "scenarios": [{
    "name": "My First Scenario",
    "model_approach": "competitive",
    "years": [{
      "year": "2030",
      "total_cap": 300.0,
      "auction_mode": "explicit",
      "auction_offered": 180.0,
      "price_lower_bound": 0.0,
      "price_upper_bound": 200.0,
      "banking_allowed": false,
      "participants": [
        {"name": "Power", "initial_emissions": 200.0, "free_allocation_ratio": 0.2,
         "penalty_price": 200.0, "abatement_type": "linear", "max_abatement": 80.0, "cost_slope": 1.5},
        {"name": "Steel", "initial_emissions": 120.0, "free_allocation_ratio": 0.4,
         "penalty_price": 200.0, "abatement_type": "linear", "max_abatement": 36.0, "cost_slope": 2.5}
      ]
    }]
  }]
}

Run it:

python ets_framework.py --config my_first_scenario.json

You should see an equilibrium carbon price of 48.75 and these per-firm results:

ParticipantAbatement (Mt)Residual emissions (Mt)Allowances bought (Mt)
Power32.5167.5127.5
Steel19.5100.552.5

Reading it: the price settles where total purchases (127.5 + 52.5 = 180 Mt) exactly absorb the 180 Mt auctioned. Power, with the flatter MAC (cost_slope 1.5), abates more than Steel (2.5). That's the whole engine in miniature: one price, each firm on its own cost curve.

Anatomy of the fields

FieldMeaning
total_capTotal allowances the system permits this year (Mt CO₂e).
auction_offeredAllowances sold at auction. The clearing solves price so demand = this.
free_allocation_ratioShare of a firm's baseline given free (reduces what it must buy).
initial_emissionsBusiness-as-usual emissions before any abatement (Mt). The baseline.
max_abatement, cost_slopeLinear MAC: marginal cost rises with slope up to a cap.
penalty_priceCost of non-compliance — the de-facto price ceiling per firm.
Try itHalve Steel's cost_slope to 1.25 and re-run. Steel now abates more cheaply, so total abatement rises and the clearing price falls. Then set auction_offered to 140 (a tighter market) and watch the price climb.

2Add a policy mechanism — the MSR

The Market Stability Reserve (Kollenberg & Taschini 2019) is a supply valve: it withholds auction volume into a reserve when the banked surplus is too large, and releases it when the bank runs short. Turn it on with scenario-level fields (it needs multiple years and banking to do anything):

{
  "name": "With MSR",
  "model_approach": "competitive",
  "msr_enabled": true,
  "msr_upper_threshold": 60.0,
  "msr_lower_threshold": 10.0,
  "msr_withhold_rate": 0.15,
  "msr_release_rate": 20.0,
  "years": [ /* multiple years, banking_allowed: true … */ ]
}

Each year, before clearing: if the total bank > msr_upper_threshold, withhold msr_withhold_rate × auction volume into the reserve; if the bank < msr_lower_threshold, release msr_release_rate Mt back. Watch the MSR Withheld, MSR Released and MSR Reserve Pool columns in the output.

Run the worked examplepython ets_framework.py --config examples/climate_solutions_msr_stability.json — two scenarios, identical market, MSR on vs off. Full details in docs/multi-year-simulation.md.

3An adaptive cap — the Carbon Cap Rule new

The Carbon Cap Rule (Benmir, Roman & Taschini 2025) is the cap-and-trade analogue of a Taylor rule. Instead of a fixed cap, it nudges each year's permit quantity using the previous year's deviation of emissions and abatement cost from reference (steady-state) levels:

Q_t = Qbar + phi_e · (e − ebar)/ebar  +  phi_z · (z − zbar)/zbar

with phi_z > 0 (loosen the cap when abatement costs run hot) and phi_e < 0 (tighten when emissions overshoot). Enable it scenario-side:

{
  "name": "Carbon Cap Rule (CCR)",
  "model_approach": "competitive",
  "ccr_enabled": true,
  "ccr_phi_emissions": -30.0,
  "ccr_phi_abatement_cost": 15.0,
  "ccr_reference_emissions": 424.0,
  "ccr_reference_abatement_cost": 2600.0,
  "years": [ /* … */ ]
}

Run the bundled comparison (fixed cap vs CCR, under a persistent shock):

python ets_framework.py --config examples/benmir_ccr_carbon_cap_rule.json

The CCR loosens the cap while abatement costs are elevated, holding the post-shock price peak down and cutting price volatility by ≈27%:

YearFixed cap — priceCCR — priceCCR cap adjustment (Mt)
202657.957.90.0
202768.770.6−2.5
2028113.4110.4+3.9
2029128.4105.7+29.4
2030122.3103.5+24.3

See the formula, sign conventions, and config reference in docs/carbon-cap-rule.md.

Try itSet ccr_phi_abatement_cost to 0 (respond to emissions only) and re-run. The cap stops chasing the cost spike and the 2029–30 prices climb back toward the fixed-cap path — showing how much of the CCR's stabilising power comes from the cost term.

4Feedback A — the price-elastic baseline new

So far the baseline (initial_emissions) is fixed: the only price response is abatement. Feedback A adds the missing channel — carbon-intensive activity contracts as the price rises ("demand destruction"), solved jointly with the price inside clearing:

activity multiplier   m(P) = max(0, 1 − ε · (P − P_ref)/P_ref)
effective baseline    E(P) = initial_emissions · m(P)

Switch it on with a scenario anchor price plus a per-firm elasticity:

{
  "name": "Price-Elastic Baseline",
  "model_approach": "competitive",
  "reference_carbon_price": 55.0,
  "years": [{
    "year": "2030", "total_cap": 480.0, "auction_mode": "explicit", "auction_offered": 216.0,
    "price_lower_bound": 0.0, "price_upper_bound": 400.0, "banking_allowed": false,
    "participants": [
      {"name": "Steel",  "initial_emissions": 180.0, "free_allocation_ratio": 0.3, "penalty_price": 400.0,
       "abatement_type": "linear", "max_abatement": 54.0, "cost_slope": 2.6, "output_price_elasticity": 0.5},
      {"name": "Power",  "initial_emissions": 220.0, "free_allocation_ratio": 0.2, "penalty_price": 400.0,
       "abatement_type": "linear", "max_abatement": 99.0, "cost_slope": 1.5, "output_price_elasticity": 0.5},
      {"name": "Cement", "initial_emissions": 120.0, "free_allocation_ratio": 0.3, "penalty_price": 400.0,
       "abatement_type": "linear", "max_abatement": 30.0, "cost_slope": 3.0, "output_price_elasticity": 0.5}
    ]
  }]
}

The bundled example runs this across a tightening cap (480 → 330 Mt) next to an inelastic twin:

python ets_framework.py --config examples/feedback_a_price_elastic_baseline.json

Where the inelastic market lets the price escalate, the elastic baseline lets firms cut output instead — so the price path stays far flatter:

YearFixed baseline — priceElastic baseline — price
2026122.874.2
2027132.576.9
2028143.980.1
2029156.983.8
2030171.588.0
How to see the contraction.In the participant results, the Initial Emissions column now reports the price-scaled baseline. In 2030 Steel's 180 Mt baseline shows up as ≈126 Mt — activity fell ~30% under the high price. Set output_price_elasticity to 0 (or reference_carbon_price to 0) to recover the inelastic model exactly.
Try itThe elastic baseline is a restoring force toward reference_carbon_price. Raise it to 120 — closer to where the inelastic price sat — and the two paths converge: there's less "distance" for activity to respond to. This is reduced-form, own-price response; for cross-sector and macro feedback, continue to Step 5. Details: docs/feedback-price-elastic-baseline.md.

5Feedback B — couple to an external model new

Feedback A keeps everything inside the allowance market. Feedback B goes further: it iterates the ETS to a joint equilibrium with a separate model — an energy-system dispatch, a CGE, a DSGE — through a thin adapter. The ETS stays the specialist allowance-market engine; the wider economy lives behind the adapter. This is a programmatic (Python) workflow rather than a single JSON run:

from ets.coupling import run_coupled_simulation, ElasticityExternalModel
import json

config = json.load(open("examples/feedback_a_price_elastic_baseline.json"))
# use just the inelastic scenario as the baseline the external model responds to
config = {"scenarios": [s for s in config["scenarios"]
                        if s["name"] == "Fixed Baseline (inelastic)"]}

result = run_coupled_simulation(
    config,
    ElasticityExternalModel(elasticity=0.5, reference_price=55.0),  # the reference stand-in model
    max_iterations=40,
    tolerance=0.25,      # converge when max |Δprice| ≤ 0.25
    relaxation=0.5,      # under-relax to damp oscillation
)

print("converged:", result.converged, "in", result.iterations, "iterations")
print(result.summary[["Year", "Equilibrium Carbon Price"]])

Or just run the bundled demo:

python examples/feedback_b_coupling_demo.py

The loop solves p = ETS(model.respond(config, p)) as a fixed point. Feeding the price back into activity drives the same demand destruction as Feedback A — but as an outer loop, and it converges in a handful of iterations:

Iteration2030 price (being solved)

Notice the coupled answer (≈78 → 100 across the years) lands close to Feedback A's in-engine result (≈74 → 88) — two routes to the same idea. The relaxation factor is what tames the 171 → 51 → 89 → 100 oscillation you can see settling above.

Write your own external model

Any object with a respond method is a valid adapter. Swap the reference elasticity model for a real one — here, a sketch wrapping an energy-system dispatch:

import copy

class EnergySystemModel:
    def respond(self, baseline_config, prices, iteration):
        cfg = copy.deepcopy(baseline_config)
        for scenario in cfg["scenarios"]:
            for year in scenario["years"]:
                co2_price = prices[(scenario["name"], year["year"])]
                dispatch = my_energy_model.solve(co2_price=co2_price)   # your model here
                for p in year["participants"]:
                    p["initial_emissions"] = dispatch.emissions[p["name"]]
        return cfg

run_coupled_simulation(config, EnergySystemModel())
Why not just embed a CGE/DSGE?Keeping the boundary explicit preserves the tool's transparency and lets you reuse a purpose-built, separately-calibrated economy. The adapter contract, the under-relaxation maths, and a runnable reference adapter are documented in docs/feedback-coupling.md.
Try itSet relaxation=1.0 (undamped) and re-run the demo — with a strong elasticity the iteration oscillates and may not settle. Then bring it back to 0.5 and lower it further to 0.3: convergence is slower but rock-steady. This is the classic speed-vs-stability trade-off of fixed-point coupling.
